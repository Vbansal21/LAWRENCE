# HMM Dynamic User Interaction Adaptation

This directory seeds the analytics and operational documentation layer for LAWRENCE.

Purpose:
- capture interaction hypotheses and adaptation signals,
- keep MCP/Skills/Tools/API docs near runtime data,
- maintain legible notes/logs/journals/workflows as first-class artifacts.

Subdirectories:
- `documents`: concept docs, state models, observation schemas.
- `MCP`: MCP integration templates and active plans.
- `Skills`: skill structure templates and current roadmap.
- `Tools`: tool contracts and capabilities.
- `APIs`: API specs and endpoint contracts.
- `notes`: short, linked operational notes.
- `logs`: append-only event logs.
- `journals`: periodic synthesis and reasoning journals.
- `workflows`: workflow definitions and templates.
